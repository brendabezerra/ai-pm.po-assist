# Email Draft Response Prompt

## Instructions

Draft a reply to the given email. Follow these guidelines:

### Tone Matching
- Read the original email's tone (formal, semi-formal, casual) and match it
- If the sender uses greetings/sign-offs, include them
- If the sender is direct and brief, be direct and brief

### Structure
1. **Acknowledge**: Brief acknowledgment of what was asked/shared
2. **Answer/Action**: Direct response or commitment to action
3. **Next step**: Clear next step or call to action
4. **Sign-off**: Appropriate closing

### Rules
- Keep under 5 paragraphs unless the topic requires detail
- If you need more information to respond, draft a reply that asks for it
- If the email requires a decision, present options with a recommendation
- Never commit to deadlines without the user confirming first — use "I'll confirm the timeline and get back to you" as placeholder
- Reference the project context when relevant
- If multiple topics are in one email, use bullet points or numbered items

### Format
```
Subject: Re: [original subject]

[Greeting],

[Body]

[Sign-off]
```

## User Review
Always present the draft for user review before sending. Highlight any parts that need the user's input (dates, decisions, specific numbers).
