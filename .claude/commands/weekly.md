# /weekly — Week Planning

You are running the weekly planning workflow. Follow the steps in `workflows/weekly_planning.md`.

## Instructions

1. Read `rules/agent_rules.md` and `rules/communication_style.md` for behavioral guidelines.
2. Read all active project context files in `projects/` (skip `_template/`).
3. Execute the weekly planning workflow:

### Past Week Review
- Check for completed action items and deliverables
- Identify carried-over items that weren't completed
- Note key decisions made

### Calendar Analysis
- Use Google Workspace MCP to fetch next week's events
- Identify heavy meeting days and available focus time
- Flag meetings that need preparation

### Project Status
- Review each active project's context and sprint status
- Identify at-risk deliverables or overdue items

### Focus Areas
- Generate **Top 5 Focus Areas** for the week
- Map each to specific days based on calendar availability
- Suggest prep schedule for key meetings

## Output
Produce the full weekly planning briefing following the format in `workflows/weekly_planning.md`.
